# Разработĸа архитеĸтуры и детальное проеĸтирование

Наш сервис будет использоваться для составления списка интересующих пользователя книг, поэтому ожидаемая R/W нагрузка составляет 80/20. Также немало важным будет учитывать объем трафика на нашем сервисе. 
Ожидается, что средний трафик в сутки будет составлять 10000 пользователей, что достаточно реалистичное число. Исходя из требований к платформе, на которой будет устанавливаться сервис и минимальных требований к техническому оснащению пользователей было принято решение о установке периода хранения информации сроком 2 года. 
Далее важно уточнить размер дискового пространства для нашего сервера, где будут хранить все данные. Оно будет составлять 4 Гб, так как ограничения для каждого пользователя по добавлению книг с свою библиотеку будет 1000, размер одной книги составляет 100 байт. Если 10000 пользователей, то размер дискового пространства считается как 100байт * 1000 книг * 10000 пользователей + 520(размер одного аккаунта) * 10000 пользователей, округляя получаем 4 Гб.
